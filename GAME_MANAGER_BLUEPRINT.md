## 데이터 구조 (CSV 컬럼)

| 컬럼 | 설명 | 예시 |
|------|------|------|
| name | 게임명 | 하데스 2 |
| status | playing / completed / paused / dropped / wishlist | playing |
| slot | AAA / AA / A / - | AA |
| genre | 장르 | 로그라이크, 액션 |
| hours | 총 플레이 시간 | 18 |
| satisfaction | 만족도 1-5 | 4.5 |
| comment | 한 줄 코멘트 | 엔딩 보고 업적까지 다 팔 게임 |
| start_date | 시작일 YYYYMMDD | 20260323 |
| end_date | 완료일 YYYYMMDD | |
| price | 구매가격 | 29900 |
| priority | 위시리스트 우선순위 high/mid/low | high |

## 화면 구성 (Tkinter)

### 탭 1: 현재 플레이 [Playing]

```
┌─────────────────────────────────────────────┐
│  AAA               AA              A         │
│  ┌─────────────┐  ┌─────────────┐  ┌──────┐ │
│  │발더스게이트3│  │  하데스 2   │  │발라트│ │
│  │  80h        │  │   18h       │  │로    │ │
│  │[+시간] [완료]│  │[+시간] [완료]│  │[+시간│ │
│  └─────────────┘  └─────────────┘  └──────┘ │
└─────────────────────────────────────────────┘
```

기능:
- 슬롯별 현재 게임 카드 표시
- [+시간] 버튼: 플레이 시간 추가 입력
- [완료] 버튼: 만족도 + 코멘트 입력창 → completed로 이동
- [일시정지] 버튼: paused로 이동

### 공통 기능

**[+ 게임 추가] 버튼:**
- 이름만 입력
- status 선택 (playing / wishlist)
- 나머지는 나중에 Claude가 채워줌

**[CSV 내보내기] 버튼:**
- game_db.csv 저장
- Claude에게 공유 → 장르, slot, 우선순위 자동 분류 후 돌려받기

---

## Claude와의 협업 워크플로우

```
1. 게임 추가 (이름만 입력)
      ↓
2. CSV 내보내서 Claude에게 공유
      ↓  
3. Claude가 장르, slot, 우선순위, GOTY 후보 여부 채워서 CSV 반환
      ↓
4. 프로그램에서 CSV 다시 불러오기
      ↓
5. 게임 완료 시 만족도 + 코멘트 입력
      ↓
6. 필요할 때 Claude에게 공유 → 추천, 분석
```
